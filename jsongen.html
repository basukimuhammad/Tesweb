Tujuan dan Konsep JSO (Javascript Overlay):
JSO atau Javascript Overlay adalah teknik yang digunakan untuk mengenkripsi atau menyembunyikan kode HTML di dalam JavaScript. Cara kerjanya adalah dengan mengonversi teks HTML ke dalam format yang bisa dievaluasi kembali menjadi HTML saat dijalankan oleh browser. Ini berguna dalam skenario di mana kamu ingin menyembunyikan atau melindungi kode HTML yang sensitif atau kompleks dari akses langsung oleh pengguna.

Komponen Script JSO:
<script> Tag: Tag ini digunakan untuk memuat skrip JavaScript di dalam halaman HTML. Dalam konteks JSO, ini adalah titik masuk di mana skrip JSO akan dimuat dan dieksekusi.
type="text/javascript": Atribut type menentukan jenis konten yang ada dalam tag <script>, yaitu JavaScript. Ini memastikan bahwa konten yang dimuat dari URL yang diberikan adalah skrip JavaScript yang akan dieksekusi oleh browser.
src="https://jso-tools.z-x.my.id/raw/~/U7GFCL6NAPOVA": Atribut src menunjukkan URL dari mana browser harus mengunduh skrip JavaScript. Dalam kasus ini, URL yang diberikan mengarah ke alamat di mana skrip JSO yang berisi HTML yang telah dienkripsi tersedia untuk diambil.

Proses Kerja:
Enkode HTML ke JSO: Sebelum skrip JSO ini dapat dibuat, HTML yang ingin disembunyikan atau dienkripsi harus diubah menjadi format JSO. Proses ini umumnya melibatkan enkode teks HTML ke dalam bentuk yang dapat dievaluasi kembali (misalnya, menggunakan Base64 atau metode enkode khusus lainnya).
Pemuatan dan Eksekusi: Ketika browser menemui <script> dengan src yang ditetapkan ke URL JSO, ia mengunduh skrip dari server. Setelah diunduh, skrip ini dieksekusi. Dalam konteks JSO, skrip ini akan mengevaluasi teks HTML yang terenkripsi dan mengubahnya kembali menjadi struktur HTML yang sesuai, kemudian menempatkannya di dalam halaman web.

Penggunaan Umum:
Penggunaan JSO umumnya ditemukan dalam pengembangan web yang memerlukan keamanan tambahan terhadap kode HTML yang sensitif. Ini juga dapat digunakan untuk mengelola dan memuat konten dinamis atau tindakan yang kompleks tanpa perlu memaparkan detail implementasinya secara langsung di dalam kode sumber halaman web.
